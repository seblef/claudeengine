# RoadTool — In-Scene Spline Editing (Issue #718)

## Summary

Adds `RoadTool`, an `EditorToolBase` subclass for live in-scene editing of
`GameRoad` spline control points. The tool auto-activates when a `GameRoad` is
selected and deactivates when the selection changes.

## RoadTool behaviour

### OnActivate
Reads the currently selected object from `ctx.scene->GetSelectedObject()`,
static-casts to `game::GameRoad*` (after type check), and caches the pointer.

### OnDeactivate
Resets all state: `road_`, `selected_point_`, `dragging_`, `hovering_`.

### OnEvent (Delete key)
When `selected_point_ >= 0` and the spline has more than 3 points, calls
`spline.RemoveControlPoint(selected_point_)` and clamps `selected_point_` to
the new count. Minimum of 3 control points is enforced.

### OnRender
1. **Width slider overlay** — `ImGui::Begin("##road_overlay")` pinned to
   top-left of the viewport via `SetNextWindowPos`. `SliderFloat` over
   `[1, 30]` m calls `road_->SetWidth()` and `Regenerate()` on change.
2. **Point projection** — `ProjectPoint()` uses `Vec4f * vp` to go from world
   to NDC, then to screen. Points behind the camera (w ≤ 0) are skipped.
3. **Hover detection** — iterates all points, tracks the closest one within
   8 px; sets `hovering_` accordingly.
4. **Circles** — `ImDrawList::AddCircleFilled` with radius 6 px; yellow for
   selected, white for hovered, grey otherwise.
5. **ImGuizmo gizmo** — translate gizmo drawn at the selected point every
   frame. On transform change: `spline.SetControlPoint()` + `Regenerate()`.
   `dragging_` mirrors `ImGuizmo::IsUsing()`.
6. **Click handling** — on LMB release without gizmo usage:
   - If hovered point: select it.
   - Else: `ComputeTerrainHit()` → `spline.InsertControlPoint()` +
     `Regenerate()`. New point becomes selected.

### IsCapturingMouse
Returns `dragging_ || hovering_` to block the selection tool.

## Auto-activation (EditorWindow)

In `EditorWindow::Render()`, after the sculpt-tool block (section 11e):

```cpp
if (!play_mode_->AreToolsFrozen()) {
    const bool road_selected =
        sel && sel->GetType() == GameObjectType::kRoad;
    if (road_selected && active_tool != EditorTool::kRoad) {
        toolbar_->SetActiveTool(EditorTool::kRoad);
        viewport_->SetActiveTool(road_tool_.get());
    } else if (!road_selected && active_tool == EditorTool::kRoad) {
        toolbar_->SetActiveTool(EditorTool::kSelection);
        viewport_->SetActiveTool(selection_tool_.get());
    }
}
```

The transition block also handles `kRoad`:
```cpp
else if (active_tool == EditorTool::kRoad)
    viewport_->SetActiveTool(road_tool_.get());
```

## Decisions

- **No toolbar button** — `kRoad` is not in any `IsCreationTool` /
  `IsTransformTool` check; the toolbar renders no button for it.
- **kMinPoints = 3** — deletion guard consistent with the issue spec.
- **height_fn = nullptr allowed** — `GameRoad::RegenerateMesh(nullptr)` is
  documented as valid (flat road following control-point Y values).
- **Overlay uses NoInputs flag** — the slider still works because `NoInputs`
  only prevents mouse passthrough when the cursor is over the window; the
  slider receives events via ImGui's own input routing.

## Kept in mind for follow-up

- Undo/redo for spline edits (noted as M3 work in the issue).
- The Delete key handler is in `OnEvent()` rather than `OnRender()` to keep
  keyboard input decoupled from the render path; this matches the intent in the
  issue spec even though `TerrainSculptTool` uses only `OnRender`.
