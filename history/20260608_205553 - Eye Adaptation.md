# Eye Adaptation (Auto-Exposure)

**Date:** 2026-06-08  
**Issue:** #413  
**Branch:** feat/eye-adaptation

---

## Summary

Implemented eye adaptation (auto-exposure) via iterative log-luminance downsampling to a 1×1 R16F render target, followed by temporal smoothing of the `exposure` value in `PostProcessInfos`.  When `PostProcessConfig::IsEyeAdaptationEnabled()` is `true`, the computed exposure replaces the fixed `1.0` default each frame.

## Algorithm

### Luminance downsample chain

Each frame (when eye adaptation is enabled):

1. **Pass 0 (`lum_init`):** The HDR RGBA16F buffer is converted to log-luminance into `levels_[0]` (W/2 × H/2, R16F):
   ```
   L = dot(rgb, vec3(0.2126, 0.7152, 0.0722))   // BT.709
   out = log(max(L, 1e-4))                        // log-space, floor avoids -inf
   ```
2. **Passes 1 … N (`lum_downsample`):** Each level halves the previous with a 4-tap bilinear average, reaching 1×1.

3. **CPU readback:** `glReadPixels` on the 1×1 FBO reads `avg_log_lum`.

4. **Exposure update:**
   ```
   avg_lum    = exp(avg_log_lum)
   target_exp = 0.18 / avg_lum           // middle-grey key
   t          = 1 - exp(-dt * adapt_speed)
   exposure   = lerp(prev, target, t)     // temporal smoothing
   exposure   = clamp(exposure, 0.1, 10)  // prevent extreme flicker
   ```

### First frame guard

`first_frame_` is `true` until the first `Update()` call, which returns `1.0f` immediately (no GPU passes) to avoid reading garbage from uninitialised textures.

### Temporal smoothing

The standard `1 - exp(-dt * speed)` formulation is frame-rate independent: the lerp factor is proportional to elapsed time, so the speed of adaptation is not tied to frame rate.

## Abstract layer extension

A new pure virtual `ReadPixelF(RenderTargetGroup* fbo, int x, int y)` was added to `abstract::VideoDevice` to read a single float pixel.  The GL implementation uses `glReadPixels` after binding the FBO as `GL_READ_FRAMEBUFFER`.  `GLRenderTargetGroup::GetFboId()` exposes the GL FBO name to `GLVideoDevice` (same pattern as `RenderTarget::GetNativeHandle()`).

## Decisions

- **R16F** for all luminance levels: single-channel float, half-bandwidth of RGBA16F, sufficient for log-luminance range (log(0.0001) ≈ −9.2 to log(100+) ≈ 4.6).
- **4-tap average in log-space** (not linear): averaging in log-space approximates a geometric mean of luminances, which is perceptually closer to how the human visual system adapts.
- **CPU readback of 1×1 pixel**: necessary to update `PostProcessInfos::exposure` (a CPU-side struct uploaded to the UBO). The cost of a 1×1 `glReadPixels` is negligible.
- **`kKey = 0.18`** (middle grey): industry standard for auto-exposure.
- **`[0.1, 10.0]` clamp** matches the issue requirement and avoids pathological values in pure black or very bright scenes.
