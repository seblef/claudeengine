## Design decisions

### Shader location — `terrain/` sub-folder

The issue specifies `data/shaders/glsl/terrain/terrain_vs.glsl`.  This matches the existing pattern (`geometry/`, `lighting/`) and keeps terrain shaders isolated from the flat root.  The GLShader loader accepts arbitrary slash-separated names so `CreateShader("terrain/terrain")` resolves correctly.

### `textureLod(..., 0.0)` in vertex shader

GLSL requires an explicit LOD level when sampling in a vertex shader (the derivative-based implicit LOD is undefined outside fragment shaders). All five heightmap samples use `textureLod(..., 0.0)` to ensure correct, portable behaviour.

### Varyings: `v_world_pos`, `v_world_normal`, `v_world_uv`

The PS currently uses only `v_world_normal`, but `v_world_pos` and `v_world_uv` are forwarded in anticipation of splatmap blending (issue #292) and future lighting features. The small varying overhead is negligible.

### Central-difference normal step size

`mpt = patch_scale / (1 << lod_level)` always equals `meters_per_texel`, making the normal derivation texel-aligned regardless of the current LOD level. This keeps normal quality uniform across LOD transitions.

### Placeholder albedo

`out_albedo = vec4(0.4, 0.5, 0.3, 0.0)` matches the issue specification. Full splatmap blending is deferred to issue #292 (Phase 3).

## Output to keep in mind for future features

- **Issue #292 (splatmap)**: `v_world_uv` (normalised 0–1 world UV) and `v_world_pos` are already passed to the PS and ready to use for texture lookups.
- The `terrain_patch_infos.glsl` UBO (binding 6) was already added in #286; it is unchanged here.
- Shader path convention: `CreateShader("terrain/terrain")` → `terrain/terrain_vs.glsl` + `terrain/terrain_ps.glsl`. Any new terrain shader variant should follow the same sub-folder pattern.
