# SoundManager (#665)

## Summary

Added `audio::SoundManager` and `audio::VirtualSoundInstance` вЂ” a virtual source pool that multiplexes an unbounded number of logical sound events onto a fixed set of hardware `ISoundSource` objects each frame, with distance culling and priority-based ranking.

## Update() algorithm (per frame)

1. **Cleanup** вЂ” iterate instances; if `state == kStopped` or (non-loop + source + `!IsPlaying()`): return source to pool, release Sound ref, erase.
2. **Cull** вЂ” for each remaining instance compute `distВІ = pos.SquaredDistance(listener_pos)`. If `distВІ > max_distanceВІ`: pause+return source, mark `kSuspended`, skip.
3. **Score** вЂ” `score = (priority + 1) / (distВІ + 1)`. Collect into a `std::vector<Candidate>` and `std::sort` descending. The `+1` offset ensures priority-0 sounds are still distance-ranked.
4. **Release excess** вЂ” for candidates[pool_size..end]: if they hold a source, pause it and return to pool; mark `kSuspended`.
5. **Assign** вЂ” for candidates[0..min(N,pool_size)]: if no source, pop from free list, `SetBuffer/Position/Gain/Pitch/Loop/Play`, mark `kPlaying`. If already has source, just update position/gain.
6. **Tick backend** вЂ” `sound_system_->Update(dt)`.

## Key design decisions

**`std::list<unique_ptr<VirtualSoundInstance>>`**
`std::list` gives O(1) erase in step 1 without invalidating any other element's pointer. `unique_ptr` wraps the private-constructor instance so `SoundManager` (a `friend`) can call `new VirtualSoundInstance(...)` directly while callers only ever see raw non-owning `VirtualSoundInstance*` handles.

**Private constructor + `friend class SoundManager`**
Prevents callers from allocating instances on the stack or heap directly. `SoundManager::PlaySound` is the single factory. `emplace_back` cannot access a private constructor through template forwarding, so explicit `new` inside the friend is used.

**Score = (priority+1) / (distВІ+1)**
The `+1` offset in both numerator and denominator avoids:
- Division by zero when `distВІ = 0` (listener at emitter)
- Score = 0 for priority-0 sounds (they are still distance-ranked)

**Suspend = restart, not resume**
When a suspended instance re-acquires a source, `Play()` starts from the beginning of the buffer. There is no seek. This is acceptable for looping ambient emitters (the loop point restarts cleanly) and one-shot effects that survived culling. Documented in CLAUDE.md.

**Pool pre-allocated at construction**
All `pool_size` `ISoundSource` objects are created upfront. `real_sources_` owns them via `unique_ptr`; `free_sources_` holds raw pointers to the available subset. Assigned instances hold a raw pointer while in use.

## Usage example

```cpp
audio::SoundManager mgr(sound_system, 32);

// Each frame:
mgr.SetListenerTransform(camera_matrix);
mgr.Update(dt);

// Fire-and-forget one-shot:
Sound* s = resources.LoadSound("explosion");
mgr.PlaySound(s, hit_pos);
s->Release();

// Looping with handle:
Sound* amb = resources.LoadSound("wind");
VirtualSoundInstance* h = mgr.PlaySound(amb, emitter_pos, true, 1);
amb->Release();
// each frame:
h->SetPosition(moving_pos);
// cleanup:
h->Stop();  // handle invalid after next Update()
```

## Output to keep in mind

- `SoundManager::SetListenerTransform()` must be called before `Update()` every frame.
- `VirtualSoundInstance*` handle is invalidated after `Stop()` + `Update()` or natural completion; do not dereference after `IsStopped()` returns true.
- `SoundManager` destructor calls `StopAll()` вЂ” safe to let it go out of scope.
- Future `GameSoundEmitter` (#666) should hold a `VirtualSoundInstance*` and call `SetPosition()` in its world-transform update hook.
