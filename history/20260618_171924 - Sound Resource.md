# Sound Resource (#663)

## Summary

Added `Sound` as a first-class reference-counted resource type in the `audio` module, consistent with `GameMaterial` and `MeshTemplate`. A `Sound` holds decoded PCM audio uploaded to the active `ISoundSystem` backend, plus all playback metadata parsed from a `.sound.yaml` template file.

| `src/audio/SoundDesc.h` | POD descriptor: `file`, `loop`, `priority`, `rolloff`, `min_distance`, `max_distance`, `volume`, `pitch` |

| `data/sounds/example.sound.yaml` | Annotated YAML template for sound assets |

## Decoder integration

All decoders are pulled into a single `AudioDecoderUtils.cpp` translation unit (the "one `*Utils.cpp`" rule for free functions):

| Format | Library | Availability |
|---|---|---|
| WAV | dr_wav (FetchContent) | Always |
| MP3 | dr_mp3 (FetchContent) | Always |
| FLAC | dr_flac (FetchContent) | Always |
| OGG/Vorbis | stb_vorbis (existing stb FetchContent) | Always |
| OPUS | opusfile (system pkg-config) | Optional — `HAVE_OPUSFILE` flag |

`dr_wav`, `dr_mp3`, and `dr_flac` are all header-only and their `*_IMPLEMENTATION` macros are defined in `AudioDecoderUtils.cpp` only, following the standard single-TU pattern.

`stb_vorbis.c` is included directly (it is its own implementation, not just a header). The `short**` output parameter requires `reinterpret_cast<short**>` with a `// NOLINT(runtime/int)` suppression since the API is C and we cannot change it.

All decoders output signed 16-bit PCM (`kMono16` / `kStereo16`), which is the most compatible format with OpenAL and future backends.

## Key design decisions

**Placed in `audio` module, not `game`**  
The `Sound` resource is a pure audio concept (it only needs `ISoundSystem*`). Placing it in `audio` keeps the dependency graph clean (`game → audio → abstract → core`) and avoids making the game module responsible for audio decoding logic.

**YAML path convention: `data/sounds/{name}.sound.yaml`**  
Consistent with `data/materials/{name}.yaml` used by `GameMaterial`. The `file` field inside the YAML is relative to the data folder, allowing audio assets to live anywhere under `data/`.

**OPUS support is optional**  
`libopusfile-dev` is not installed on the build machine. The feature is guarded by `#ifdef HAVE_OPUSFILE`, enabled automatically when `pkg_check_modules(OPUSFILE opusfile)` succeeds — same pattern as `HAVE_LIBTIFF`. No functionality is lost when absent.

**Always decode to 16-bit**  
This simplifies the `AudioFormat` mapping (only mono/stereo distinction remains) and is compatible with all current and planned backends. dr_libs and stb_vorbis all provide `_s16` output functions.

## Usage example

```cpp
// Load (or cache-hit) a sound
Sound* snd = audio::Sound::GetOrLoad("explosion", sound_system);

// Attach to a source and play
auto src = sound_system->CreateSource();
src->SetBuffer(snd->GetBuffer());
src->SetGain(snd->GetDesc().volume);
src->Play();

// Release when done
snd->Release();
```

## Output to keep in mind

- `Sound::GetOrLoad()` / `Release()` must be balanced, same as `GameMaterial`.
- `Sound::GetDesc()` exposes the full `SoundDesc` for callers that need to apply `rolloff`, `min_distance`, `max_distance`, or `pitch` to a source — the `Sound` class itself does not configure sources.
- Future work: a `GameSound` wrapper (game module, like `GameMaterial`) could tie a `Sound` to a `GameObject` transform and auto-configure `ISoundSource` spatialization using `SoundDesc` fields.
