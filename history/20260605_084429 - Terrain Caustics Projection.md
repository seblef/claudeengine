# Terrain Caustics Projection (#359)

## What was done

Implemented animated caustic light projection onto submerged terrain fragments, producing the bright rippling vein patterns seen on the seabed when sunlight is focused through a water surface.

## Changes

### `src/environment/WaterRenderer` (h + cpp)
- Added `BuildCausticTexture()` private method that generates a 256×256 RGBA8 tileable caustic texture procedurally at `Build()` time.
- The texture is generated by summing six cosine ring wavefronts emanating from irregularly-placed source points. **Toroidal (wrap-around) distance** is used so that all circular patterns tile seamlessly. Values above a threshold of 4/6 of the signal range become bright veins; below is dark background.
- Added `caustic_tex_` (`unique_ptr<RawTexture>`) member and `GetCausticTexture()` accessor.
- Added `caustic_tex_.reset()` to `Reset()` for clean resource release.

### `src/terrain/TerrainRenderer` (h + cpp)
- Added `SetCausticTexture(abstract::RawTexture*)` setter and `caustic_tex_` pointer member.
- Added `kCausticSlot = 11` constant (one above the existing `kMacroTextureSlot = 10`).
- In `Render()`, the caustic texture is bound to slot 11 when set (null-checked).

### `src/renderer/Renderer` (h + cpp)
- Moved `SetTerrainRenderer()` from inline in `Renderer.h` to `Renderer.cpp` so it can access `WaterRenderer::GetCausticTexture()`.
- Both `SetWaterRenderer()` and `SetTerrainRenderer()` now cross-wire the caustic texture regardless of which renderer is registered first.

### `data/shaders/glsl/terrain/terrain_ps.glsl`
- Added `#include <uniforms/scene_infos.glsl>` (provides `time` for UV scrolling animation).
- Added `#include <uniforms/water_infos.glsl>` (provides `water_params.a` = water level and `refraction_params.y` = absorption_scale used as depth falloff rate).
- Added `layout(binding = 11) uniform sampler2D u_caustic_tex`.
- After albedo computation, applied caustic brightening to submerged fragments:
  - **Submersion test**: `world_y < water_params.a`
  - **Two scrolling UV layers** sampled from the caustic texture and multiplied together — the product emphasises bright peaks and suppresses uniform areas, creating more dynamic interference.
  - **Depth falloff**: `exp(-water_depth * refraction_params.y)` mirrors the water absorption calculation.
  - **Brightening**: `albedo *= 1.0 + caustic * falloff * 2.0`.

## Decisions and rationale

- **Slot 11 for caustic texture**: Slots 0–10 are fully occupied (heightmap, splatmap, 4 albedos, 4 normals, macro). Slot 11 is the next free slot.
- **Toroidal distance in caustic generation**: Arbitrary source-point positions would not tile unless the circular ring pattern wraps around the texture boundary. Clamping the dx/dz components to `(-size/2, +size/2]` achieves seamless tileability.
- **Multiplying two scrolling caustic samples**: A single scrolling sample animates but lacks variation. Two layers scrolling in opposite directions create a more naturalistic interference pattern where bright veins cross and merge.
- **Reusing `absorption_scale` as depth falloff**: The issue specified this directly; it also makes physical sense — the same absorption coefficient that attenuates light traveling upward through the water column also attenuates sunlight focusing downward.
- **Same `terrain_ps.glsl` for tessellated and non-tessellated passes**: Both the standard VS path and the TESE path output the same `v_world_pos` varying, so no separate PS was needed.
- **Cross-wiring in both `SetWaterRenderer` and `SetTerrainRenderer`**: Map loading can set the terrain renderer before or after the water renderer (e.g., editor vs. game), so both directions of wiring are needed.

## What to keep in mind for next features

- `u_caustic_tex` (slot 11) is **always declared** in the terrain shader even when no caustic texture is bound — OpenGL will sample from a default 1×1 white texture in this case, which is harmless (caustic = 1 × 1 = 1, falloff = 0 since world_y ≥ water_lvl for dry land). The bind guard in `Render()` is a micro-optimisation for GPU state changes.
- The caustic texture is a static precomputed grayscale pattern — all animation comes from the UV scrolling. A future improvement could animate the actual texture (e.g., pre-baked flipbook or compute shader).
- `refraction_params.y` (`absorption_scale`) is shared between the water absorption calculation and the caustic depth falloff. If the two ever need to diverge, a dedicated `caustic_falloff` field should be added to `WaterInfos`.
