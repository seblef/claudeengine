# Texture Support (Issue #43)

## Summary

Added a complete texture pipeline: `TextureFormat` enum, abstract `Texture` resource class, OpenGL `GLTexture` implementation with stb_image (JPEG/PNG) and optional libtiff (TIFF) loading, a `CreateTexture` factory on `VideoDevice`, updated shaders that sample from a 2D texture, and a fullscreen-quad demo rendering `data/textures/demo.png`.

## Design Decisions

### Texture as a Resource (CRTP)
`Texture` inherits from `core::Resource<std::string, Texture>` — the same registry pattern as `Shader`. This means `Texture::Get(name)` returns `Texture*` directly without casts, and `AddRef()`/`Release()` handle shared ownership. Loading the same file twice returns the cached instance.

### width_/height_ initialized to 0, set after loading
The base constructor is called with `0, 0` dimensions; `GLTexture` sets `width_` and `height_` after stb_image returns the actual values. The protected members in `Texture` allow this without friend declarations.

### stb_image flip
`stbi_set_flip_vertically_on_load(1)` corrects OpenGL's bottom-left origin for image files stored top-to-bottom. UVs are set accordingly: TL=(0,1), BL=(0,0), BR=(1,0), TR=(1,1).

### TIFF via libtiff — conditional compilation
`find_package(TIFF QUIET)` provides graceful degradation: TIFF files log an error and return uninitialized if libtiff is absent. `TIFFGetField` returns `uint32_t*` for width/height, cast to `int` for storage.

### stb_image_impl.cpp
Placing `STB_IMAGE_IMPLEMENTATION` in a dedicated .cpp avoids multiple-definition link errors — a common pitfall with single-header libraries.

### sampler2D binding = 0
`layout(binding = 0) uniform sampler2D` requires GL 4.2+ (engine targets GL 4.6). The texture is bound to slot 0 each frame via `tex->Bind(0)` before `RenderIndexed()`.

## For Future Features

- `TextureFormat` is defined but not yet used for `glTexImage2D` format selection — currently all textures are uploaded as `GL_RGBA8`. A `ToGLFormat(TextureFormat)` helper will be needed when supporting formats beyond RGBA8.
- `stbi_set_flip_vertically_on_load` is a global flag; if multiple threads load textures concurrently, this will need a mutex or per-call alternative.
- The `stb` FetchContent uses `GIT_TAG master` (latest), matching the project's convention for loguru. Pin to a specific commit/tag if build reproducibility becomes a concern.
