## Design decisions

### Jolt-free public headers
`PhysicsVehicle.h` uses forward declarations (`namespace JPH { class VehicleConstraint; class Body; class PhysicsSystem; }`) to keep all `#include <Jolt/...>` in the `.cpp` files, matching the existing module contract.

### Private constructor + PhysicsSystem friend
Same pattern as `PhysicsBody`: only `PhysicsSystem` can construct `PhysicsVehicle` objects. All Jolt object creation (body, constraint, step listener registration) happens inside `PhysicsSystem::CreateVehicle()` in `PhysicsSystem.cpp`.

### Ref counting for VehicleConstraint
`VehicleConstraint` is ref-counted. `PhysicsVehicle` takes ownership via `AddRef()` in the constructor and `Release()` in the destructor, matching how `PhysicsBody` manually manages `base_shape_`.

### Driver input via PreStepCallback
A lambda capturing `this` is installed on the constraint via `SetPreStepCallback`. It calls `WheeledVehicleController::SetDriverInput()` at the start of each sub-step, so inputs set by game code any time before `PhysicsSystem::Step()` are applied at the correct simulation moment.

### Destructor order
Vehicle destructor does:
1. `RemoveStepListener` ŌĆö stops the constraint from running during `Update()`
2. `RemoveConstraint` ŌĆö releases Jolt's ownership ref; constraint destructor runs while body is still alive
3. `Release()` ŌĆö releases our ownership ref ŌåÆ constraint memory freed
4. `RemoveBody` + `DestroyBody` ŌĆö removes and frees the body

`PhysicsSystem::~PhysicsSystem()` clears `vehicles_` before `bodies_` and before `jolt_system_.reset()` to ensure vehicles are torn down while Jolt is still valid.

### Differential setup
`CreateVehicle()` inspects `WheelDesc::is_driven` on the front and rear pairs and adds a `VehicleDifferentialSettings` entry per driven axle. Torque is split evenly across driven axles (50/50 for AWD).

### Suspension spring
`WheelDesc::suspension_stiffness` and `suspension_damping` map to `JPH::SpringSettings(ESpringMode::StiffnessAndDamping, k, c)` so the physical constants are used directly.

### Wheel transform convention
`GetWheelWorldTransform(i)` returns a matrix that positions a cylinder model aligned with its local Y axis, using `inWheelRight = Vec3::sAxisX()` and `inWheelUp = Vec3::sAxisY()` (Jolt's default wheel orientation).

### Collision tester
`VehicleCollisionTesterRay` against `kLayerWorld` for wheel-ground detection (terrain/static geometry).

## Output to keep in mind

- Wheel indices are: 0 = front-left, 1 = front-right, 2 = rear-left, 3 = rear-right.
- Hand-brake torque is applied only to rear wheels (indices 2 and 3).
- `GetBodyWorldTransform()` returns the body origin transform (not COM), suitable for rendering the vehicle mesh.
- For wheels to steer, `WheelDesc::is_steered = true` must be set and `VehicleDesc::max_steer_angle` specifies the limit in radians.
- The next step (issue #712 or similar) will wire `PhysicsVehicle` to a `GameVehicle` entity in the `game/` layer.
