# DDS Compressed Texture Support (Issue #590)

## Summary

Added support for DDS block-compressed textures (BC1/BC3/BC5/BC7) in the OpenGL texture loading pipeline via the GLI header-only library.

## Changes

### `CMakeLists.txt`
- Added **GLM 0.9.9.8** (header-only math, required by GLI) via FetchContent.
- Added **GLI master branch** (header-only DDS/KTX loader) via FetchContent.
  - Note: GLI's stable tag `0.8.2.0` is incompatible with GLM 0.9.9+ due to a template template parameter issue (old API used `tvec2<T,P>` with 2 params; GLM 0.9.9+ uses `vec<L,T,Q>` with 3 params). The `master` branch has this fixed: `template <length_t L, typename T, qualifier Q>` in `levels.inl`.

### `src/gldevices/CMakeLists.txt`
- Added `${gli_SOURCE_DIR}` and `${glm_SOURCE_DIR}` to gldevices' private include directories.

### `src/gldevices/GLTexture.h`
- Updated class comment to document DDS/KTX compressed format support.

### `src/gldevices/GLTexture.cpp`
- Added `#include <gli/gli.hpp>` wrapped in pragma guards to suppress GLI's benign enum-conversion deprecation warning in `save_dds.inl`.
- Added a compressed loading block at the top of the constructor for `.dds`, `.ktx`, and `.ktx2` extensions:
  - Loads via `gli::load()` ŌåÆ validates target is `TARGET_2D` ŌåÆ constructs `gli::texture2d`.
  - Translates format via `gli::gl(gli::gl::PROFILE_GL33).translate()` to obtain the `GL_COMPRESSED_*` internal format and channel swizzles.
  - Sets `GL_TEXTURE_BASE_LEVEL` / `GL_TEXTURE_MAX_LEVEL` so OpenGL knows the mip chain bounds.
  - Sets `GL_TEXTURE_SWIZZLE_R/G/B/A` from GLI's swizzle map ŌĆö critical for single/dual-channel formats like BC4 and BC5.
  - Uploads each mip level with `glCompressedTexImage2D` (no `glGenerateMipmap` needed ŌĆö mipmaps are pre-baked in the DDS file).
- Existing PNG/JPEG/TIFF path is unchanged and runs only when the extension is not a compressed format.

### `data/textures/test_bc1.dds`
- Minimal 4├Ś4 BC1/DXT1 DDS file (136 bytes) generated by a Python script.
- Solid-red 4├Ś4 block; serves as a smoke-test asset to verify the load path at runtime.

## Design Decisions

- **No changes to `abstract::Texture` or `TextureFormat`**: compressed formats are an implementation detail of the GL backend. The CPU never touches individual pixels, so there is no value in surfacing `kBC1` etc. through the abstract interface.
- **GLI master vs 0.8.2.0**: the latest stable tag is incompatible with GLM 0.9.9+. Using `master` is consistent with how `loguru`, `stb`, `fast_obj`, and `ufbx` are pinned in this project.
- **Channel swizzles**: applied via `glTexParameteri(GL_TEXTURE_SWIZZLE_*)` for correctness with BC4/BC5 which only store R or RG channels.
- **Mip filter**: when the DDS has more than 1 mip level, `GL_LINEAR_MIPMAP_LINEAR` is used; otherwise `GL_LINEAR`, matching the intent of a DDS without mipmaps.

## Keep in Mind

- DX12 backend (future): DXGI formats map 1:1 to BCn, so DDS support there will be straightforward.
- KTX2 is included in the extension check but requires GLI master's KTX2 support to be complete; this was not explicitly tested.
- GLM is now a transitive build dependency of gldevices but is not used outside of GLTexture.cpp.
