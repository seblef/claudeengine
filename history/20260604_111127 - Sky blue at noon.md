# Sky blue at noon — issue #380

**Branch**: `fix/sky-yellow-noon-380`
**Issue**: #380
**Date**: 2026-06-04

---

## Problems identified

### 1. Fixed warm sun disc and halo colours (PR #392, merged first)

The sun disc used constant warm colours (`vec3(60, 48, 24)`, ratio 5:4:2) and
the halo used `vec3(2.0, 1.6, 0.9)`, both regardless of sun elevation.
At noon the sun should appear near-white. Fixed in `sky_ps.glsl` with an
elevation-dependent blend (orange at horizon → near-white above 15°).

### 2. Wrong azimuth direction (this commit)

Both `WorldTime::GetSunDirection()` and `SkyRenderer::ComputeSunDirection()`
had negated x/z components:
```cpp
// WRONG — sun rises west, is north at noon
const float x = -std::cos(azimuth) * cos_el;
const float z = -std::sin(azimuth) * cos_el;
```
This placed the sun to the **north** at noon (should be south for NH) and made
it rise in the **west** (should be east). The Mie forward-scattering peak in
the Preetham model concentrated warm colours in the wrong directions, making
the sky appear yellow when looking where the sun actually was.

### 3. Hardcoded latitude with no way to tune it

The model hardcoded latitude 45° N on equinox (noon sun elevation = 45°,
solar zenith angle θs = 45°). The Preetham model at θs = 45° gives a
pale blue sky. Using a lower latitude or summer solstice declination raises
the sun significantly: at lat 45° N with summer solstice (δ = +23.45°), noon
elevation is ≈ 68°, θs ≈ 22°, giving a noticeably deeper blue sky.

### 4. Debug LOG_F left in SkyRenderer::Render() (removed)

A `LOG_F(INFO, "sun dir: ...")` was left in the hot render path.

## Changes

### `src/environment/WorldTime.h`

- Replaced simplified arc model with proper hour-angle formula documentation.
- Added `latitude_deg_` (default 45) and `declination_deg_` (default 0) members.
- Added `SetLatitude()`, `SetDeclination()`, `GetLatitude()`, `GetDeclination()`.

### `src/environment/WorldTime.cpp`

- Extracted `ComputeSunDir(hour, lat, decl)` free function with the standard
  astronomical formula:
  ```
  sin(α) = sin(φ)·sin(δ) + cos(φ)·cos(δ)·cos(H)
  cos(A) = (sin(δ) − sin(φ)·sin(α)) / (cos(φ)·cos(α))
  ```
- Fixed the azimuth: removed the two stray negation signs; sun now rises east,
  transits south (NH) at noon, sets west.
- Engine coordinates (+X east, +Y up, +Z south):
  `x = sin(A)·cos(α)`, `z = −cos(A)·cos(α)`.
- Removed the `kMaxElevationRad` constant; elevation is derived from lat/decl.

### `src/environment/SkyRenderer.h`

- Made `ComputeSunDirection` non-static (accesses `latitude_deg_` and
  `declination_deg_` members).
- Added `latitude_deg_` (default 45) and `declination_deg_` (default 0).
- Added `SetLatitude()` and `SetDeclination()` with documentation noting they
  must match the values set on `WorldTime` for consistent lighting.

### `src/environment/SkyRenderer.cpp`

- Implemented `ComputeSunDirection` with the same astronomical formula as

- Removed the `loguru.hpp` include (no longer needed).
- Removed stale constants `kSunriseHour`, `kSunsetHour`, `kMaxElevationRad`.

## Decisions

### Same formula in both WorldTime and SkyRenderer

WorldTime drives scene lighting (global light direction, shadows) and
SkyRenderer drives the sky shader sun disc. Both must agree on the sun
direction or the sun disc in the sky won't match the shadow direction.
They cannot share code at the link level (environment module must not depend
on renderer), so the formula is duplicated; the free `ComputeSunDir` helper
in WorldTime.cpp makes the intent clear.

### Latitude and declination as runtime setters, not YAML fields

The sky colour depends strongly on these values and the user wants to tune
them interactively. Adding them to the map YAML is a future follow-up;
for now they default to 45° N equinox (identical behaviour to before,
except the sun is now in the correct direction).

## Output to keep in mind

- Call `world_time.SetLatitude(d)` **and** `sky_renderer.SetLatitude(d)` (same
  value) when changing latitude so scene lighting and sky match.
- Summer solstice (`SetDeclination(23.45f)`) raises the noon sun from 45° to
  ≈ 68° at latitude 45° N, giving a noticeably bluer Preetham sky.
- The sun now rises in the east and transits through the south at noon for
  northern latitudes; existing map orientations may need to be checked.
