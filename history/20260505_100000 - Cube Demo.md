# 3D Cube Demo (Issue #53)

## Summary

Assembled all engine pieces into a real 3D interactive demo: a textured cube rotating at 20°/s around the Y axis, rendered with proper depth testing, two constant buffer UBOs for world and camera matrices, and a first-person camera controlled by keyboard and mouse.

## Design Decisions

### row_major UBO layout
`Mat4f` is stored row-major in C++. GLSL `mat4` defaults to column-major. Uploading raw `Mat4f::Data()` with the default column-major layout would transpose the matrix. Using `layout(std140, row_major, binding=N)` tells the GPU to read memory row-by-row, so GLSL `M * v` gives the same result as the C++ `v * M` (column-vector convention). No explicit transpose needed in C++.

### UBO binding namespaces
`layout(binding=0)` for a `sampler2D` (texture unit 0) and for a UBO block (uniform buffer binding point 0) occupy separate GL namespaces — no conflict.

### view_proj = proj * view
`Camera::UpdateMatrices()` computes `view_proj_ = proj_ * view_`. In the shader: `view_proj * world * pos` = `proj * view * world * pos`. ✓

### 24-vertex cube (no shared vertices across faces)
Each face needs its own set of vertices to have distinct normals, tangents, and UV coordinates. Sharing 8 vertices would produce incorrect normals and UVs at corners. 4 vertices × 6 faces = 24; 2 triangles × 3 indices × 6 faces = 36.

### SetDepthTestEnabled on the abstract interface
Rather than calling GL directly in main.cpp (which breaks abstraction), depth test control was added to `abstract::VideoDevice`. This is a natural engine primitive: any 3D rendering needs depth test, any 2D/UI overlay disables it.

### Camera yaw/pitch from mouse deltas
Absolute mouse position events are received. Previous position is tracked; the first event sets the baseline without changing orientation (avoids a jump). Pitch is clamped to ±1.5 rad (≈±85°) to avoid gimbal issues near vertical.

### Key held-state pattern
Events are edge-triggered (KeyDown/KeyUp). Boolean flags track which keys are currently held so per-frame camera translation can use delta time correctly.

## For Future Features

- `textured_mesh` shader does not use normals/tangents/binormals. When lighting is added, these are already in the vertex stream — just extend the VS to output them to the FS.
- Camera speed (20 units/s) and mouse sensitivity (0.002 rad/px) are compile-time constants in the anonymous namespace. These should move to config.yaml when a proper settings system is in place.
- The demo keeps `TODO(#2)` for the Engine class — once that's implemented, the demo scaffolding in main.cpp should be migrated into an Engine-managed scene.
