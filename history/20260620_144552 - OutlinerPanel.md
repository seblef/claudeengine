# feat(editor): OutlinerPanel — scene hierarchy tree view (#658)

## What was built

A new `OutlinerPanel` class that replaces the "Objects" tab in the Scene side-panel with a proper DFS hierarchy view. Three supporting undoable commands were added.

### Commands

| File | Purpose |
|------|---------|

## Decisions and rationale

### Placement: `src/editor/` (not `src/editor/panels/`)

The issue spec mentions `src/editor/panels/` but all existing panels (`ObjectsPanel`, `PropertiesPanel`, etc.) live directly in `src/editor/`. Creating a half-populated subdirectory would be inconsistent. The file lives in `src/editor/` to match every other panel.

### ObjectsPanel kept alongside OutlinerPanel

`ObjectsPanel` is preserved in the codebase; only the "Objects" tab in `EditorWindow` is replaced by the new "Outliner" tab. This allows future reference and avoids destroying working code in a single PR. The old tab can be removed in a follow-up.

### DeleteSubtreeCommand: bottom-up reclaim, top-down undo

`EditorScene::ReclaimDynamicObject` calls `DetachChildren` (reparents obj's children to obj's parent) and then `Reparent(nullptr)`. By processing the subtree in DFS bottom-up order (leaves first), each node's children list is already empty when it is reclaimed, so `DetachChildren` is always a no-op. On undo, nodes are re-added top-down so parents exist in the scene before children are reparented back to them.

### Non-dynamic descendants are skipped

`DeleteSubtreeCommand::Collect` guards with `IsDynamic(obj)`. The only non-dynamic object is the global directional light, which is never parented to a user-created pivot; this guard is a safety net.

### ReparentCommand captures old_parent_ at construction

The constructor snapshots `obj->GetParent()` immediately. This is safe: the command is created synchronously inside the drag-drop accept path or a context menu callback, before any other mutation.

### Ctrl+click multi-select bypasses SetSelectedObject

Standard click calls `scene.SetSelectedObject(obj)` (clears existing selection). Ctrl+click calls `AddToSelection` / `RemoveFromSelection` to toggle membership, matching typical DCC tool conventions.

### Drop-to-root: InvisibleButton below all nodes

After the tree, an `ImGui::InvisibleButton` sized to the remaining content region acts as a drag-drop target for "move to root". Dropping onto it calls `ReparentCommand(dragged, nullptr)`.

### Pivot icon: folder open/closed only when children exist

Empty pivots show the crosshairs icon (like other objects); the folder icon is reserved for pivots that have at least one child — this avoids a confusing "empty folder" that cannot expand.

## Output to keep in mind

- `OutlinerPanel` replaces the "Outliner" tab content; `ObjectsPanel` is still compiled but no longer rendered by default.
- If `ObjectsPanel` is to be removed entirely, update `EditorWindow.h/.cpp` and `CMakeLists.txt` in a follow-up.
- `AddChildPivotCommand` stores a raw `game::GameObject* parent_` pointer. If the parent is deleted before Redo is called, the pointer is dangling. In practice, Undo always restores the parent to the scene before Redo can re-parent under it, so this is safe within the current command-history model.
- The drag-drop type tag `"OUTLINER_OBJ"` is local to the outliner; it will not conflict with viewport drag-drop sources.
