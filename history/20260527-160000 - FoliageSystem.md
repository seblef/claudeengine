# Foliage System — GPU Instanced Rendering with Billboard LOD

**Date:** 2026-05-27  
**Branch:** `feat/foliage-system-298`  
**Issue:** #298  

---

## Summary

Implements a complete terrain foliage scattering system with GPU instanced mesh rendering (near) and billboard LOD (far), including a density-brush editor UI and YAML serialisation.

## Key Design Decisions

### SSBO for per-instance data
SSBOs have a separate binding namespace from UBOs, so SSBO binding 0 (near matrices) and binding 1 (billboard vec4s) coexist safely with UBO bindings 1–6.

### CPU culling per frame
Each frame, `ClassifyAndUpload()` iterates all instances for a layer and splits them into near (distance ≤ billboard_distance) and billboard (> billboard_distance, ≤ cull_distance) lists. These are uploaded to two `kDynamic` SSBOs and then drawn with a single `glDrawElementsInstancedBaseVertex` each. Capped at `kMaxFoliageInstances = 32768` per layer.

### Instance generation: jittered grid
Instances are placed on a `spacing_max × spacing_max` grid with a random jitter within each cell. Density map [0, 255] determines the probability of placing an instance in each cell (0 = never, 255 = always). The seed is fixed at 42 for deterministic results. Scale and yaw are randomised per instance.

### Billboard design
Cylindrical billboards (fixed world-up Y) are used rather than spherical, to avoid tipping over for tall foliage like trees. The billboard VS reads the instance `vec4(pos.xyz, scale)` from SSBO[1] and expands a 1×1 quad using `view[0].xyz` (right) and `vec3(0,1,0)` (up). Rendered in the emissive pass at 0.7× brightness to avoid appearing brighter than lit geometry.

### Density map format
Saved as raw `.r8` binary (single-channel uint8, row-major, same dimensions as terrain heightmap). Avoids stb dependencies in terrain/game modules; read/write with `std::fstream`.

### Billboard auto-generation deferred
The issue requested offscreen-RT auto-generated billboard textures. This is deferred to a future feature; billboards use the same `texture_path` albedo texture as near meshes.

## SSBO Binding Table

| Slot | Data | Usage |
|---|---|---|
| 0 | `mat4 world[]` (64 bytes each) | Near foliage instance world matrices |
| 1 | `vec4 pos_scale[]` (16 bytes each) | Billboard instance positions + uniform scale |

- `FoliageRenderer::Instance()` requires the singleton to be created by `GameTerrain::OnAddedToScene()` — the `Renderer::SetFoliageEnabled()` toggle safely no-ops if `FoliageRenderer` is not ready
