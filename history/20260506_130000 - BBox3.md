## Key Decisions

### Union as `operator<<` (in-place)

The union operator extends the box in-place and returns `*this&`, like `<<=` would â€” allowing chaining: `box << a << b << c`. No copy is made, which is efficient for building an enclosing box from a stream of points or sub-boxes.

### Matrix Transform (8-corner method)

`operator*(Mat4f)` transforms all 8 corners of the box through the matrix and recomputes the AABB. This is the correct conservative approach for any affine transform â€” a simple corner transform preserves the tightest possible axis-aligned box. The initial result box is seeded from corner 0 (both min and max), then `<<` extends it for the remaining 7.

### `IsCompletelyIn` â€” radius on Vec3f only

`IsCompletelyIn(const BBox3&)` â€” no radius: two boxes either nest or they don't.  
`IsCompletelyIn(const Vec3f&, float radius)` â€” radius shrinks this box inward, useful for proximity/sphere-in-box tests.

### Ray Intersection (slab method, scalar inv_dir)

## Output / Things to Keep in Mind

- **No CMakeLists.txt change**: header-only, included directly where needed.
- **`operator<<` is mutating**: unlike typical stream operators, this one modifies `*this`. Document clearly at call sites.
- **Uninitialized default**: `BBox3()` leaves `min_` and `max_` uninitialized (matching `Vec3f` convention). Always set both before use.
- **`IsCompletelyIn(BBox3)` check direction**: `rhs` is the box being tested for inclusion inside `*this`. Parameter order matches natural English: "is `rhs` completely in `*this`?"
