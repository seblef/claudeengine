## Overview

Implements `WindSystem`, a time-varying wind simulation that exposes a per-frame XZ wind vector consumed by GPU shaders via a new `WindInfos` constant buffer at slot 7.

## Design decisions

### WindInfos struct layout (`vec2 + float + float = 16 bytes`)

The GLSL spec requires `vec2` at std140 alignment 8. Using `core::Vec2f wind_xz` (2 plain floats, no special alignment, 8 bytes) followed by two `float` fields gives exactly 16 bytes = 1 float4, zero padding needed. A `static_assert(sizeof(WindInfos) == 16)` guards this.

### Y component always zero

`WindSystem::GetWindVector()` returns `{wind_vec_.x, 0.f, wind_vec_.z}` explicitly, even though `wind_vec_` is computed from a zero-Y `base_direction_`. This keeps the invariant self-documenting and robust against future code that might modify `wind_vec_` directly.

### Phase accumulator is in seconds, not radians

`phase_` accumulates raw seconds; the frequency scaling `phase_ * 2π * gust_frequency_` happens inside `Update()`. This avoids precision loss from accumulating large radian values directly at higher frequencies and makes resetting or inspecting the phase easier.

### Renderer always binds slot 7

`wind_infos_cb_->Bind()` is called unconditionally alongside the other global CBs at the top of `Renderer::Update()`. When no `WindSystem` is set, `FillWindInfos()` uploads all-zero data. This ensures shaders that read slot 7 always see a valid (if idle) buffer rather than undefined state.

### Renderer does not advance the wind

The caller is responsible for calling `WindSystem::Update(dt)` each frame. The renderer only reads the current state and uploads it to the GPU. This matches the pattern for other environment systems (e.g. the renderer does not advance `WorldTime` either).

## Output / notes for future features

- Slot 7 is now globally bound. Water, cloud, and foliage shaders can `#include "uniforms/wind_infos.glsl"` to read `wind_xz`, `wind_strength`, and `wind_time` without any per-shader setup.
- `gust_amplitude_` and `gust_frequency_` are private with compile-time defaults (0.3, 0.2 Hz). If per-descriptor wind tuning is needed, add these fields to `EnvironmentDesc` and wire them through the constructor.
- `wind_time_` accumulates forever and will lose sub-second precision after ~194 days. This is acceptable for a game engine; if needed, wrap at a large period (e.g. `fmod(wind_time_, 86400.f)`).
