## Key Decisions

### std140 Packing (`SceneInfos`)

Five `Mat4f` fields occupy the first 320 bytes. The tail is carefully laid out to match std140 alignment rules:

| Offset | Field | Type | Note |
|--------|-------|------|------|
| 320 | `eye_pos` | `Vec3f` | 16-byte slot (Vec3f is 12 bytes, padded by std140) |
| 336 | `time` | `float` | 4 bytes |
| 340 | `pad0_` | `float` | **Explicit** — `vec2` needs align-8, so next field must be at offset 344 |
| 344 | `inv_screen_size` | `Vec2f` | 8 bytes |
| **352** | end | | 22 float4s |

`static_assert` verifies all offsets and total size at compile time, so any future drift causes an immediate build error.

### `time_` as Member

`FillSceneInfos()` takes no parameter — it reads `time_` from the `Renderer` member. This avoids a subtle bug: if `SetCamera` were called mid-frame, it would invoke `FillSceneInfos` and the time would be lost (reset to 0 or an old value). By storing `time_` as a member updated only in `Update()`, `SetCamera` correctly re-uses the current time.

### Shader Layout

The full `SceneInfosBlock` (all 22 float4s) is declared in the vertex shader even though only `view_proj` is consumed. This is required by the GL driver — the UBO binding size must match what the driver sees. Fields not consumed are simply ignored by the shader compiler.

## Output / Things to Keep in Mind

- **Slot convention**: Slot 0 = per-object world matrix, Slot 1 = per-frame `SceneInfos` (always owned by `Renderer`). All future shaders binding slot 1 must declare the full `SceneInfosBlock`.
- **`FetchContent` / CMake**: `renderer` links `PUBLIC abstract` so downstream targets (like `app`) transitively get `abstract` headers.
- **`inv_screen_size` computation**: `camera_->GetScreenCenter()` returns `(half_w, half_h)`, so `inv_screen_size = {1/(2*half_w), 1/(2*half_h)} = {1/width, 1/height}` — correct for texture-space reconstruction.
- **`pad0_` is explicit**: Do not remove it even if a compiler happens to insert it automatically — explicit padding makes the intent clear and prevents future regressions if the struct is reordered.
