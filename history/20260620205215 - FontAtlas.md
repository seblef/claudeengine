# FontAtlas — stb_truetype integration (#692)

## What was added

### `abstract::VideoDevice::CreateAtlasTexture`
New pure-virtual factory method:
```cpp
[[nodiscard]] virtual std::unique_ptr<RawTexture> CreateAtlasTexture(
    int width, int height, const uint8_t* data) = 0;
```
Returns a single-channel R8 texture with `GL_CLAMP_TO_EDGE` wrapping and `GL_LINEAR` filtering — optimal properties for font sampling. No mipmaps.

### `gldevices::GLAtlasTexture`
Concrete `abstract::RawTexture` subclass (new files `GLAtlasTexture.h/.cpp`).  Uses `GL_R8` internal format; supports `UpdateRegion()` for partial re-uploads (future dynamic atlases).

### `ui::FontAtlas` (`src/ui/FontAtlas.h/.cpp`)
High-level class that wraps the full pipeline:
1. Reads a TTF file from disk via `std::ifstream`
2. Bakes printable ASCII (codepoints 32–126) with `stbtt_BakeFontBitmap` — tries 512×512 first, falls back to 1024×1024
3. Uploads the R8 bitmap via `VideoDevice::CreateAtlasTexture`
4. Computes line height from font vertical metrics

Public API:
```cpp
bool Load(abstract::VideoDevice*, const char* path, int pixel_height);
void Destroy();
const GlyphInfo* GetGlyph(char c) const;
abstract::RawTexture* GetTexture() const;
int GetLineHeight() const;
```

`stb_truetype.h` is already present in `external/stb-src/`; `STB_TRUETYPE_IMPLEMENTATION` is defined in `FontAtlas.cpp` (single TU).

### New CMake target `ui`
`src/ui/CMakeLists.txt` — static library linking `abstract`; includes `${stb_SOURCE_DIR}` privately. Added to `src/CMakeLists.txt`.

## Decisions and rationale

| Decision | Rationale |
|---|---|
| `RawTexture*` instead of `Texture*` in the public API | `abstract::Texture` is reference-counted and registry-keyed by filename. A CPU-baked atlas has no file identity and must be owned by the `FontAtlas` object — `RawTexture` (unique_ptr-owned) fits perfectly. |
| `GL_R8` for atlas texture | Font coverage needs only 1 byte per texel. Using a single-channel format avoids a 4× memory cost compared to RGBA8, keeps bandwidth low, and is straightforward to sample in shaders. |
| No new `VideoDevice::CreateAtlasTexture` on R8 specifically | The abstract interface must not know about GL internals, but it's correct to add a semantic method rather than repurposing `CreateNormalMapTexture` (which is RGBA8 and semantically for terrain). |
| 512×512 with 1024×1024 fallback | At pixel_height ≤ 48 all 95 ASCII glyphs fit in 512×512. The fallback handles large sizes or high-density fonts without hardcoding a worst case. |
| `kFirstChar`/`kNumGlyphs` public | The anonymous-namespace helper `TryBake` needs access; making them public also documents the baked range for callers. |

## Output to keep in mind

- **`ui` library is created but not linked by any executable yet.** Consumers (`game`, `editor`) will need to add `target_link_libraries(... ui)` when they use `FontAtlas`.
- **Shader side not covered.** The atlas texture exposes a single red channel (`GL_R8`). Text rendering shaders must read `texture(atlas, uv).r` for coverage and multiply by the desired text colour.
- **No kerning.** `stbtt_BakeFontBitmap` does not include kerning tables. A future enhancement could use `stbtt_GetCodepointKernAdvance`.
- **`abstract/VideoDevice.h` gained a pure-virtual method.** Any future backend (e.g., DirectX 12) must implement `CreateAtlasTexture`.

## Instructions especially taken into account
- One class per `.h`/`.cpp` file
- Google C++ style guide
- Include root is `src/`
- New module needs `CMakeLists.txt` and entry in `src/CMakeLists.txt`
- `#define STB_*_IMPLEMENTATION` in one `.cpp`
- Use `abstract::RawTexture` (not `abstract::Texture`) for CPU-uploaded data
