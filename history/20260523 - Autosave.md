# Autosave — periodic recovery save (#255)

## Context

Milestone 10: map loading & saving. Depends on #254 (File menu + dirty flag).

## Changes

### `data/config.yaml`

Added `editor.autosave_interval_seconds: 120` under a new `editor:` section.
This key is optional; if absent the editor defaults to 120 s.

### `src/editor/EditorWindow.h`

- Added `<string>` include (needed for `last_autosave_msg_` member).
- Extended the class-level comment to document the autosave contract.
- Added `void TickAutosave()` private method.
- Added four new private members:
  - `autosave_timer_` — elapsed seconds since last autosave attempt.
  - `autosave_interval_` — configurable interval (default 120 s).
  - `last_autosave_msg_` — message shown in the status bar ("Autosaved").
  - `autosave_msg_timer_` — countdown controlling message visibility (3 s).

### `src/editor/EditorWindow.cpp`

- Added `<yaml-cpp/yaml.h>`, `core/Config.h`, `core/YamlUtils.h` includes.
- Constructor: loads `data/config.yaml` and reads `editor.autosave_interval_seconds`.
- `Render()`: calls `TickAutosave()` as the very first statement each frame.
- `TickAutosave()`: accumulates `DeltaTime`; when `scene_dirty_` is true,
  the map name is non-empty and not "untitled", and the interval has elapsed,
  writes `{data}/maps/{name}.autosave.map.yaml` via `MapSerializer::Save()`.
  Logs a warning on failure (timer is still reset to avoid hammering a broken path).
  Decrements `autosave_msg_timer_` every frame.
- Status bar: renders `last_autosave_msg_` alongside the dirty indicator while
  `autosave_msg_timer_ > 0`.

## Key invariants

- Autosave writes to `{name}.autosave.map.yaml`, never to `scene_->GetFilePath()`.
- Autosave does **not** clear `scene_dirty_`.
- Autosave only triggers for named maps (not "untitled" or empty name).
- On failure, the timer resets to avoid repeated failed writes.
- The autosave file is not surfaced in any recent-files list or the Load dialog.

## Decisions

- Config is read from `data/config.yaml` directly in the `EditorWindow` constructor
  using `core::LoadYamlFile`. This avoids adding an `EditorConfig` class for a single
  scalar; if more editor settings appear later, a dedicated `EditorConfig` class
  (mirroring `GraphicsConfig`) would be the right extension point.
- `autosave_msg_timer_` is decremented in `TickAutosave()` every frame regardless
  of whether an autosave fired, so the status message fades naturally without an
  extra update path.
