# Game Audio Initialisation

**PR**: #686  
**Branch**: `fix/game-audio-init-673`

## What was done

Added audio system startup to `app/main.cpp`. The game app previously
never initialised any audio backend, so all `GameSoundEmitter` objects
in every map were silent.

## Changes

| File | Change |
|---|---|
| `app/main.cpp` | Create `ISoundSystem` / `SoundManager` / `ResourceManager`; register with `GameSystem`; pass to `MapLoader::Load`; shut down cleanly |
| `app/CMakeLists.txt` | Linked `audio` and `openal` targets |

## Root cause

`GameSystem::SetSoundManager` existed and `MapLoader::Load` already
accepted optional manager pointers, but `app/main.cpp` never wired
them up. The sound system infrastructure was built first for `GameSoundEmitter`
(#671) but the entrypoint was never updated.

## Shutdown order

`ResourceManager` holds `Sound*` references → must be destroyed before
`SoundManager` (which owns `VirtualSoundInstance` objects) → before
`ISoundSystem::Shutdown()` (which closes the device). Destroying
`sound_resources` then `sound_manager` then calling `Shutdown()` is
the correct sequence; confirmed by the OpenAL backend docs.

## Output to keep in mind

- If `SoundSystemFactory::Create()` returns nullptr (unsupported
  platform) or `Initialize()` fails, the game continues silently — no
  crash. All `GameSoundEmitter` objects stay silent, matching pre-audio
  behaviour.
- `SoundManager` and `ResourceManager` must outlive all `GameSoundEmitter`
  objects that were constructed with them. Since both are destroyed after
  `GameSystem::Shutdown()` (which calls `OnRemovedFromScene` on all
  objects), the lifetime is safe.
