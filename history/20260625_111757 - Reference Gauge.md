## Summary

Added a placeable **reference gauge**: an editor-only 1×1×1 m yellow wireframe cube placed at the camera focus point. It exists only in the editor, is saved in the map YAML under a `gauges:` key, and is never loaded or rendered at game runtime.

## Key decisions

### DrawBBoxWireframe placement
`DrawBBoxWireframe` already existed in the anonymous namespace of `PickingUtils.cpp`. Rather than exposing it in `PickingUtils.h` (which would have made the utility file aware of gauge concerns) or requiring a separate include chain, a 35-line `DrawCubeWireframe` helper was added to the anonymous namespace of `EditorViewport.cpp`. This keeps the viewport self-contained for its own overlay rendering.

### YAML placement: top-level `gauges:` key
Gauges are serialised at the top level (not inside `editor:`), matching the issue spec. The game-side `MapLoader` reads only the keys it knows about and silently ignores the rest — no changes needed there.

### Deletion: Clear All (non-undoable)
Per-gauge click-selection in the viewport was considered but is disproportionate for a first iteration. A single "Clear All Gauges" button (non-undoable) fulfils the acceptance criterion "gauges can be deleted individually or in bulk" at an acceptable complexity level.

### Undo safety
`PlaceGaugeCommand::Execute()` pushes to the back; `Undo()` pops from the back. This is safe because the command history undoes commands in strict LIFO order — when a gauge command is undone, all later commands have already been undone, restoring the vector to exactly the state it was in when Execute ran.

## Output to keep in mind

- `EditorGauge` lives in `EditorScene.h` — any code that already includes `EditorScene.h` gets `EditorGauge` for free.
- `GetCameraFocusPoint()` is now public on `EditorViewport` — useful for other one-shot placement features.
- The `DrawCubeWireframe` helper in `EditorViewport.cpp` is anonymous-namespace-local. If other systems need wireframe cube drawing, consider promoting it to `PickingUtils.h`.
- Gauges are cleared non-undoably by `ClearGauges()` — if undo support is needed later, a `ClearGaugesCommand` (saving the entire vector snapshot) can be added.
