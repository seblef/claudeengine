# Moon as a Texture — Feature Implementation

**Issue:** #442
**Branch:** feat/moon-texture
**Date:** 2026-06-10

## Summary

The moon was previously rendered as a plain white/silver circle in the sky
fragment shader. This feature adds optional texture support for the moon disc,
with elevation-based colorization in both the textured and plain-circle paths.

## Changes

### `src/environment/EnvironmentDesc.h`
Added `std::string moon_texture` field (default empty = plain circle fallback).
Also added `<string>` include since `EnvironmentDesc` was previously header-only
without string members.

### `src/environment/SkyInfos.h` + `data/shaders/glsl/uniforms/sky_infos.glsl`
Repurposed the unused `si_pad1_` slot (offset 24 in std140) as
`has_moon_texture` (float 0.0/1.0). No layout change needed — same 32-byte
struct. The shader reads this flag to choose between textured and plain paths.

### `src/environment/SkyRenderer.h` + `.cpp`
- Added `SetMoonTexture(const std::string& path)` — loads/releases textures via
  the existing `video_->CreateTexture()` resource registry.
- `Render()` now sets `has_moon_texture` in the constant buffer, binds
  `moon_tex_` to sampler slot 0 if present, and calls `UnbindSampler(0)` after
  to avoid stale bindings.
- `Reset()` releases `moon_tex_` before teardown.

### `data/shaders/glsl/sky/sky_ps.glsl`
- Added `layout(binding = 0) uniform sampler2D moon_tex`.
- Moon disc section now computes an elevation-based tint (`moon_tint`) applied
  in both paths (warm orange at horizon → cool white at zenith).
- When `has_moon_texture > 0.5`: builds an orthonormal tangent frame from
  `moon_dir`, maps angular offset to UV [0,1]², samples the texture, multiplies
  by `moon_tint`.
- Otherwise: keeps the original silver disc, now also multiplied by `moon_tint`.

### `src/game/MapLoader.cpp`
Parses `moon_texture` from the environment YAML block.

### `src/editor/MapSerializer.cpp`
Serializes `moon_texture` (only when non-empty, consistent with the other
optional fields in the block).

### `src/editor/EnvironmentEditorPanel.cpp`
- `EnableSky()` calls `SetMoonTexture(desc.moon_texture)` after `Build()`.
- `RenderSkySection()` adds "Pick moon texture" (NFD open dialog) and
  "Clear moon texture" buttons, updating `env.moon_texture` and the live
  renderer immediately.

## Design Decisions

**UV tangent frame:** The moon disc is tiny (~1.6° half-angle). Building an
orthonormal frame from `moon_dir` + world-up lets us project the view
direction's perpendicular component directly into UV space. The scale factor
`kDiscHalfRadius = 0.02828` matches `sin(acos(0.99960))` — the outer
smoothstep edge — so the texture exactly fills the visible disc.

**`has_moon_texture` as float:** Using float (0.0/1.0) rather than int avoids
GLSL integer type concerns across the std140 boundary and matches how the sun
elevation factor is stored elsewhere in the codebase.

**Colorization in both paths:** The issue asked for colorization by sky
position. Applying the warm-to-cool elevation tint to the plain circle too is
the right behavior even without a texture, making moonrise/moonset visually
richer without breaking existing scenes.

**Sampler slot 0:** The sky pass draws a fullscreen quad with no geometry
samplers active, so slot 0 is always free. `UnbindSampler(0)` after rendering
prevents interference with subsequent passes.

## Output / Notes for Future Work

- The moon texture should be an RGBA or RGB PNG placed under `data/textures/`.
  A grayscale lunar surface image works well; the engine's `CreateTexture`
  treats all files as RGBA8.
- The moon disc currently has a fixed angular size (~1.6°). If the disc radius
  becomes configurable in the future, `kDiscHalfRadius` in the shader must be
  kept in sync with the smoothstep boundaries.
- If a DX12 backend is added, `UnbindSampler(0)` must be implemented there too.
