## Design decisions

### Vertex format
Used `core::Vertex2D` (= `VertexBase`: position, color, uv) for both sprites and text.
The per-draw tint / text color is encoded in the vertex color field rather than a `SetUniform4f` call, since the abstract `Shader` interface only exposes `SetUniformFloat` and `SetUniform2f`. This avoids extending the interface and keeps UIRenderer self-contained in the `ui` library.

### Geometry
A single dynamic VBO (`kMaxQuads * 4 = 4096` vertices) and static IBO (`kMaxQuads * 6 = 6144` uint16 indices, quad pattern pre-filled once) are shared across sprites and text. One draw call per element (sprite or text element) updates the VBO and issues `RenderIndexed`.

### Orthographic projection
Standard 2D ortho: `x_ndc = 2*px/W - 1`, `y_ndc = 1 - 2*py/H`. Uploaded into CB slot 4 as a row-major `Mat4f` (matching the `row_major` qualifier in the uniform block, consistent with all other CBs in the engine).

### Render order
Sprites sorted by `layer_` first, then all texts sorted by `layer_`. This matches the issue spec ("sprites then text elements") and is simple to implement without a joint sort.

### Depth / blending state
Depth test and writes are disabled for the UI pass; alpha blending is `SrcAlpha / OneMinusSrcAlpha`. State is restored after `Render()` returns.

### Dependency direction
`ui` depends on `abstract` (and `core` transitively). `renderer` depends on `ui` (PRIVATE link). No circular dependency.

## Key decisions to remember
- CB slot 4 is exclusively the UI orthographic projection — no other pass should use it
- `ui_sprites_` and `ui_texts_` in `Renderer` are persistent lists (Add/Remove), not rebuilt each frame
- The IBO index type is `kUInt16` — `kMaxQuads * 4 = 4096` < 65535, so uint16 is safe
- UIRenderer::Render() restores depth and blend state; callers after it (currently none) see the default pipeline state
