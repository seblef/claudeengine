# CPU Profiler — Selective per-scope timing with periodic log reporting

**Issue:** #691  
**Branch:** `feat/core-cpu-profiler-691`

---

## Changes

### `src/core/Profiler.h` / `Profiler.cpp` (new)

Two new classes and a macro:

- **`ProfileSample`** — plain struct holding aggregate stats for a single named block (`avg_ms`, `min_ms`, `max_ms`, `total_ms`, `calls`).
- **`Profiler : Singleton<Profiler>`** — accumulates samples between report windows. `MarkFrame()` is called once per main-loop tick; when the configured interval elapses, `Flush()` emits a log report sorted by `total_ms` descending and resets all accumulators.  
  `GetLastReport()` / `GetLastFps()` expose the last completed window for a future HUD without any further changes to the profiler.  
  All public methods are guarded by a `std::mutex` so audio (potentially on a background thread) can call `RecordSample` safely.
- **`ProfileScope`** — RAII timer that calls `RecordSample` on destruction. Guards on `Profiler::IsInstanced()` so it is a no-op when the profiler is absent.
- **`PROFILE_SCOPE(name)`** — expands to a `ProfileScope` local with a line-number-suffixed name to allow multiple uses per scope without collision.

### `src/core/Config.h` / `Config.cpp` (modified)

Two new CLI flags:
- `--enable-profiling` — boolean presence flag; enables the profiler.
- `--profile-interval <s>` — float; report interval (default `5.0`).

The original `Config::Init` had an early `return` after `--data-path`, preventing any subsequent flag from being parsed.  Changed to a single-pass loop with no early exit.

### `src/app/main.cpp` (modified)

Three integration points:
1. After `Config::Init`: conditionally `new core::Profiler(interval)`.
2. End of each main-loop iteration (after `game.Update()`): `MarkFrame()`.
3. Before `Logger::Shutdown()`: `Profiler::Shutdown()` — ensures the final partial window is flushed before the logger closes.

### `src/core/CMakeLists.txt` (modified)

Added `Profiler.cpp` to the `core` static library, keeping the source list alphabetically ordered.

## Decisions & rationale

| Decision | Rationale |
|---|---|
| `std::mutex` instead of atomics | Simplest correct solution; the critical section is tiny (accumulate a double + compare). Avoids subtle double-check issues with atomics on min/max aggregation. |
| Sorted by `total_ms` descending | Hot paths rise to the top of the log without manual scanning. |
| Macro uses `__LINE__` suffix (`profscope_##__LINE__`) | Allows multiple `PROFILE_SCOPE` calls in the same function body without identifier collision; no leading underscore to stay cpplint-clean. |
| `ProfileScope` checks `IsInstanced()` at construction, not destruction | Avoids the race where the profiler is destroyed mid-frame while a `ProfileScope` is alive on the stack. Once `active_` is `false` the destructor is always a no-op. |
| Profiler `Shutdown()` before `Logger::Shutdown()` | Flush logs the final partial window; must happen before the log sink is torn down. |

## Output to keep in mind for future features

- **HUD integration**: `Profiler::GetLastReport()` and `GetLastFps()` are already exposed; a future `ProfilerHUD` widget can query them without touching the profiler internals.
- **`PROFILE_SCOPE`** is ready to annotate any subsystem. Suggested first sites: `AudioSystem::Update`, `Renderer::Render`, physics step, foliage pass.
- The `Config` single-pass parser fix unblocked adding further CLI flags; future additions simply add an `else if` branch.
