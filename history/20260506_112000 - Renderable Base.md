# Renderable Base Class

**Date:** 2026-05-06
**Branch:** feat/renderable-base
**Issue:** #69

## Summary

Added `renderer::Renderable`, the abstract base class for all objects that can be rendered (meshes, actors, lights, etc.). It tracks world position, maintains local and world bounding boxes, and records the last enqueue frame for visibility culling.

## Changes

### New: `src/renderer/Renderable.h` and `Renderable.cpp`

**Attributes:**
- `local_bbox_` (`core::BBox3`) — immutable local-space bounding box; set once at construction
- `world_bbox_` (`core::BBox3`) — world-space AABB; recomputed on construction and whenever the world matrix changes
- `world_matrix_` (`core::Mat4f`) — object-to-world transform
- `always_visible_` (`bool`) — bypasses frustum/occlusion culling when true
- `last_frame_` (`uint64_t`) — index of the last frame this object was enqueued; defaults to 0

**Constructors:**
- `Renderable(local_bbox, world_matrix, always_visible)` — primary constructor; computes `world_bbox_ = local_bbox * world_matrix` immediately using `core::BBox3::operator*(const Mat4f&)`, which transforms all 8 corners and returns the AABB
- `Renderable(const Renderable* other)` — copy constructor from pointer; copies all four spatial fields (per issue spec)

**Methods:**
- `Enqueue()` — pure virtual; enqueues the object for rendering in the current frame
- `GetWorldMatrix()` / `SetWorldMatrix(m)` — setter recomputes `world_bbox_`
- `GetLocalBBox()` / `GetWorldBBox()`
- `GetLastFrame()` / `SetLastFrame(frame)`
- `IsAlwaysVisible()`

Non-copyable via the copy-assign and copy-constructor deletes; the pointer-based copy constructor is the intentional duplication path.

### Modified: `src/renderer/CMakeLists.txt`

Added `Renderable.cpp` to the renderer static library sources.

## Decisions & Rationale

### `uint64_t` instead of `unsigned long`

The issue specifies `unsigned long`, but cpplint (Google C++ Style) requires fixed-width integer types. `uint64_t` is the correct equivalent on 64-bit platforms and is consistently used throughout the project.

### World bbox via `BBox3::operator*(Mat4f)`

`core::BBox3` already implements `operator*(const Mat4f&)` which transforms all 8 corners and returns their AABB. Using it directly keeps `SetWorldMatrix` and the constructor DRY.

### Copy constructor takes a pointer, not a reference

The issue explicitly asks for "copy constructor: takes a pointer to another renderable". This is an unusual but intentional design choice (possibly to allow passing nullptr to signal "no source"). The standard copy constructor (`const Renderable&`) is deleted to prevent accidental copying.
